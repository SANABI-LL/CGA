What this is& why first

An intent is the kind of question being asked — not its wording. “Where is the Reg?”, “Reg 在哪？” and “point me to Regenstein” are one intent: locate. This document fixes the small, closed set of intents the agent recognizes, and the parameters each one extracts.

It comes before the prototype because the interpretation layer can only show what it can name. Each entry below maps one-to-one to a Claude tool definition in src/agent/tools/ — its name, description, and input schema — and to a spatial operation over the self-hosted GeoJSON via Turf.js.

Seven intents cover Phases 1 through 3. Five are direct retrieval over building attributes; two are multi-step analyses that return ranked candidates, never a single “optimal” answer.

01 / UTTERANCE

Natural language

English or Chinese, free-form. “Which buildings lack elevators?”

02 / CLASSIFY

Intent + parameters

Claude tool use selects one intent and fills its input schema.

03 / EXECUTE

Spatial query

Turf.js filters / buffers / ranks the GeoJSON in the Lambda.

04 / COMPOSE

Editorial answer

Plain-English line, structured data, sourced footnote — mapped.

00

Anatomy of an intent

Every intent is one file in src/agent/tools/. The shape below is the contract between the classifier, the query engine, and the answer composer.

// src/agent/tools/locate_feature.ts
export const tool = {
  name: "locate_feature",
  intent: "locate",
  description: "Find one named place and center the map on it.",
  input_schema: {
    type: "object",
    properties: {
      name:     { type: "string", description: "Building or place, any known alias" },
      layer:    { type: "string", enum: ["buildings", "trees"] }
    },
    required: ["name"]
  },
  answer: "single_feature"  // which Editorial Answer template to render
};

03

Cross-cutting behaviour

Three rules sit above the catalog and apply to every intent.

Bilingual by default

Intent and parameters are language-agnostic. A Chinese utterance resolves to the same tool call and the same Editorial Answer, composed back in the language asked.

“Reg 在哪？”
→ locate_feature · name: Regenstein

Clarify, don’t guess

When a name is ambiguous or a required parameter is missing, the agent asks one precise question rather than returning a wrong feature. No silent assumptions.

“the science building”
→ “Which — Kersten, Crerar, or Eckhardt?”

Name the boundary

Off-dataset asks — live transit times, room booking, anything not in the campus layers — get one honest sentence stating what is out of scope. Never a fabricated answer.

“is room 122 free now?”
→ out_of_scope · no live booking layer
